**Определение:** *Хеш-таблица с открытой адресацией* - хэш-таблица, в которой все элементы лежат в самой таблице.

Коэффициент заполнения $\sigma \leq1$ 

Хэш-функция $h(k, i)$, где $k$ - ключ, а $i\in \mathbb{N}$ - номер исследования.

$\forall k$ последовательность $\{ h(k,i) \}_{i=0}^{N}$ является перестановкой $\{ i \}_{i=0}^{m-1}$
Бывает несколько стратегий:
- [[Хеш-таблица, использующая перемешивание сложением|Линейное пробирование]];
- [[Хеш-таблица, квадратичное исследование, двойное хеширование|Квадратичное пробирование]];
- [[Хеш-таблица, квадратичное исследование, двойное хеширование|Двойное хэширование]]; 
## Вставка

```c
int insert(Table t, int key, int info){
	for (int i = 0; i < t.msize; ++i){
		int = j = h(k,i) % t.msize;
		if (!t.data[j].busy){
			t.data[j].key = key;
			t.data[j].info = info;
			t.data[j].busy = 1;
			return 0;
		}
	}
	return -1;
}
```

## Поиск

```c
int search(Table t, int key){
	for (int i = 0; i < t.msize){
		int j = hash(k,i) % t.msize;
		if (t.data[j].busy == 0){
			return -1;
		}
		if (t.data[j] == key){
			return j;
		}
	}
	return -1;
}
```

## Удаление

```c
int erase(Table t, int key){
	int idx = search(t, key);
	if (idx != -1) {
		t.data[idx].busy = 0;
		return 0;
	}
	return -1;
}
```
