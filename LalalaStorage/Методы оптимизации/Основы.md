### Задача оптимизации
$$\min_{g_{i}(x)\&0, \ i=1..m, \ x\in \mathbb{Q}} f(x)$$
где $\mathbb{Q}\subseteq \mathbb{R}^{d}, f:\mathbb{Q}\to \mathbb{R}, \ \& = \ '\leq' \vee \ '='$, а $g_{i}(x): Q\to \mathbb{R}, i=1..m$ - функции, задающие ограничения.

1. В общем случае задачи могут не иметь решения
2. Задачи оптимизации часто нельзя решить аналитически
3. Сложность задачи зависит от $f, Q$  и размерности $x$.
Если задача имеет решение, то на практике её решают приближённо. Для этого применяют специальные алгоритмы, которые и называют **методами оптимизации**. 

- Эффективность метода определяется для класса задач
- Методы разрабатываются для класса задач
- Предполагается, что численный метод может накапливать информацию о задаче при помощи некоторого *оракула*. *Оракул* - некоторое устройство, которое отвечает на последовательные вопросы численного метода.
Примеры оракулов:
 1. **Оракул нулевого порядка**: возвращает в некоторой точке $x$ значение $f(x)$
 2. **Оракул первого порядка**: возвращает в запрашиваемой точке $x$ значение $f(x)$ и $\nabla f(x)$
 3. **Оракул второго порядка**: возвращает в запрашиваемой точке значение и градиент функции, а также её гессиан в данной точке $(\nabla f^{2}(x))_{ij}= \frac{\partial^{2} f(x)}{\partial x_{i}\partial x_{j}}$ 
### Общая схема метода оптимизации $\mathcal{M}$
**Входные данные**: начальная точка $x^{0}$ (0-верхний индекс), требуемая точность решения задачи $\varepsilon>0$. 
**Настройка:** Задать счётчик итераций $k=0$ и $I_{-1}=\varnothing$ (накапливаемая информационная модель этой задачи).
**Основной цикл:**
1. Задать вопрос к оракулу $\mathcal{O}$ в точке $x^{k}$
2. Пересчитать информационную модель: $I_{k}=I_{k-1}\cup(x_{k};\mathcal{O}(x^{k}))$
3. Применить правило метода $\mathcal{M}$ для получения новой точки $x_{k+1}$ по модели $I_{k}$.
### Критерии останова

- По аргументу:
$$\|x^{k}-x^{*}\|\leq\varepsilon$$
- По функции: $$f(x^{k})-f^{*}\leq\varepsilon$$
- По норме градиента: $$\|\nabla f(x^{k})\|\leq\varepsilon$$
### Сложность методов оптимизации

- **Аналитическая/Оракульная** сложность - число обращений к оракулу, необходимое для решения задачи с точностью $\varepsilon$
- **Арифметическая/Временная** сложность - общее число вычислений, необходимых для решения задачи с точностью $\varepsilon$.
