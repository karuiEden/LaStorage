### Выборка

Выборка необходима в ML, в первую очередь она нужна в выборе обучающих данных из реального мира и разбиения их на части.
Типы выборок делят на две части:
- Выборка не вероятностная;
- Вероятностная выборка.

#### Выборка не вероятностная

Это такой выбор данных, который не основан на вероятности. Существует несколько критериев такого выбора:
 - **Удобная выборка:** Образцы выбираются на основе их *доступности*. Он популярен, поэтому он *удобный*;
 - **Выборка лавиной(Snowball):** Будущие образцы выбираются на основе уже существующих образцов. Например, чтобы собрать данные с аккаунтов X, не имея доступа к БД, можно начать с малого кол-ва аккаунтов, а затем уже обрабатывать аккаунты, на которые те подписаны.
 - **Выборка жюри:** Эксперты решают, что должны включать образцы.
 - **Выборка по квоте:** Выбор образцов идет на основе квот для определенных фрагментов данных без какой-либо рандомизации.

Образцы, выбранные по не вероятностным критериям, не представляют данные реального мира, поэтому будет пронизан выбранными сдвигами.

Но такая выборка является быстрым способом, чтобы получить данные для модели с нуля.

##### Простая случайная выборка

Самая простая форма случайной выборки, имеются все образцы, равновероятные по выбору. Преимущество этого метода в простой имплементации, но недостаток этого заключается, что редкие классы будут выбраны реже.

##### Стратифицированная выборка

Чтобы избежать недостатка простой случайной выборки, можно сначала разделить данные в группы, которые вас интересуют, и проводить выборку из каждой группы по отдельности. Каждая группа называется *стратой*, а этот метод *стратифицированной выборкой*.
Один недостаток этого метода заключается в том, что он не всегда возможен, например разбить все образцы по группам.

##### Взвешенная выборка

В *взвешенной выборки*, каждый образец имеет вес, который определяет вероятность быть выбранным. Этот метод позволяет использовать экспертные знания в данной области. Это также помогает, когда данные поступают из разных распределений данных.

```python
# Choose two items from the list such that 1, 2, 3, 4 each has
# 20% chance of being selected, while 100 and 1000 each have only 10% chance.
import random
random.choices(population=[1, 2, 3, 4, 100, 1000],
weights=[0.2, 0.2, 0.2, 0.2, 0.1, 0.1],
k=2)
# This is equivalent to the following
random.choices(population=[1, 1, 2, 2, 3, 3, 4, 4, 100, 1000],
k=2)
```

Общим понятием в машинном обучении, тесно связанным с взвешенной выборкой, являются веса выборок. *Взвешенная выборка* используется для отбора выборок для обучения модели, тогда как веса выборок используются для присвоения «весов» или «значимости» обучающим выборкам. Выборки с более высокими весами оказывают большее влияние на функцию потерь. Изменение весов выборок может значительно изменить границы принятия решений вашей модели, как показано на рисунке ниже.

##### Выборка из резервуара

*Выборка из резервуара* - увлекательный алгоритм, который особенно полезен, когда нужно иметь дело с потоковыми данными, которые обычно используются в производстве.

Представьте, что есть входящий поток твитов и вы хотите образец из определенного числа, $k$, твитов, чтобы сделать анализ или обучить модель. Вы не знаете, сколько твитов будет, но вы знаете, что они все не поместятся в памяти, а это значит, что вы заранее не знаете вероятность, с которой должен быть выбран твит. Вы хотите убедиться, что:
- Каждый твит имел равную вероятность быть выбранным.
- Вы можете остановить алгоритм в любое время и все твиты выбраны с правильными вероятностью.
Одно из решений такого алгоритма:
1. Положить первые $k$ элементов в резервуар;
2. Для каждого $n^{ого}$ элемента, сгенерировать число $i:\ 1\leq i\leq n$.
3. Если $1\leq i\leq k$, то заменяем $i$-ый элемент на новый, иначе пропускаем.
Это означает, что каждый входящий $n$-ый элемент будет иметь вероятность $\frac{k}{n}$ быть выбранным.

##### Выборка по важности

*Выборка по важности* один из важных алгоритмов выборки. Он позволяет нам выбирать из распределения, когда мы имеем доступ к другому распределению.

Представьте, что вы хотите выборку $x$ из распределения $P(x)$, но оно является дорогим, медленным или невыполнимым. В то же время мы имеем распределение $Q(x)$, который намного легче для выборки. Итак, вы делаете выборку $x$ из $Q(x)$ и взвешиваете по $\frac{P(x)}{Q(x)}$. $Q(x)$ называется *предложенным распределением* или *важным распределением*. $Q(x)$ может быть любым распределением, если $Q(x)>0$ при $P(x)\neq 0$. Следующее уравнение показывает, что в среднем $x$, отобранное из $P(x)$, равно $x$, отобранному из $Q(x)$ с весом $\frac{P(x)}{Q(x)}$:
$$
E_{P(x)}[x]=\sum_{x}P(x)x=\sum_{x}Q(x)x \frac{P(x)}{Q(x)}=E_{Q(x)}\left[ x \frac{P(x)}{Q(x)} \right] 
$$

### Разметка

Бывает двух типов:
- Ручная разметка;
- Естественные метки;

#### Ручная разметка

Ручная разметка данных является довольно сложной задачей по нескольким причинам:
1. Она может быть дорогой, особенно, если область исследования требует мнения экспертов;
2. Может представлять опасность для конфиденциальности данных;
3. Ручная разметка довольно медленная. И она же делаем систему менее адаптивной.
##### Множественность меток

Чтобы получить достаточно размеченных данных компании часто используют несколько источников данных и аннотаторов. Эти различные источники данных и специалисты имеют разные степени точности. Из этого следует *проблема неоднозначности* или множественности меток.

Чтобы решить данную проблему, нужно правильно поставить задачу.

##### Происхождение данных

Бездумное использование данных из источников. сгенерированных специалистами без подтверждения их квалификации может стать причиной внезапного краха модели.

Рекомендуется отслеживать происхождение каждого из образцов данных, а также их метки - эта техника называется *прослеживаемость данных*. Прослеживаемость данных помогает вам выявить как потенциальные погрешности в данных, так и отлаживать модели.

#### Естественная разметка

Помимо ручной разметки существуют задачи с естественной разметкой. Это такие задачи, в которых прогнозы модели могут быть автоматически или частично оценены системой. Пример: длительность маршрута в приложениях карт.

Если задача не может иметь естественные метки, можно разрешить оставлять обратную связь на модель. В качестве примера можно рассмотреть приложение переводчика, в которой можно ввести альтернативный перевод.

Задачи с естественной разметкой довольно часто встречаются в индустрии.

Когда прогноз не сработал и он является плохим. То это называется *неявная метка*, так как она предполагается из-за отсутствия положительной метки. Также есть *явная метка*, когда пользователи явно демонстрируют обратную связь.

##### Длина цикла обратной связи

Для задач с естественными метками истины, время, которое проходит с момента предоставления прогноза до момента получения обратной связи по нему, называется *длиной цикла обратной связи*. В рекомендательных системах обычно она короткая, т.е. несколько минут.

Выбор правильной длины окна требует тщательного рассмотрения, поскольку он связан с
компромиссом между скоростью и точностью.
