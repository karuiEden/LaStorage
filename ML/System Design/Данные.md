### Движки хранения данных и обработка

Существует два типа обработки данных: транзакционный и аналитический.

#### Транзакционная и аналитическая обработка

В онлайн пространстве транзакциями являются любые твиты, запросы на сайтах. Они вставляются по мере генерации, иногда обновляются при надобности, а как становятся не нужны, то удаляются. Такой тип обработки называется *Online transaction processing* (OLTP).

Так как эти транзакции часто привлекают пользователей, то такие системы обработки не должны заставлять ждать пользователей, она должна обладать высокой доступностью. Это означает, что такая система должна быть доступна всегда, когда пользователь хочет сделать транзакцию. Транзакционные базы данных построчные.

Чтобы посчитать различную статистику по данным транзакционные базы данных могут быть неэффективные, поэтому существуют аналитические базы данных, они ориентированы по столбцам. Данные могут быть с различных источников. Мы называем такую обработку *online analytical processing* (OLAP).

Но данные парадигмы устарели по 3 причинам:
1. Это разделение изначально было вызвано ограничением технологий.
2. Так как эти парадигмы тесно связаны, то их можно разъединить и использовать разные движки для хранения и обработки данных;
3. Термин "онлайн" стал перегружен.

### ETL

*ETL(Extract, Transform, Load)* - принцип обработки данных.

В фазе извлечения извлекаются данные из всех источников, которые нам нужны. В данном этапе мы должны проверить данные, и не нужные нам исключить.

Обработка - главная часть, где проходит большая часть обработки данных. Это объединение данных из разных источников и очистить их. Также происходит стандартизация данных в определенном диапазоне значений. А также различные операции с данными.

Загрузка решает, как и как часто загружаются данные в место назначения.

### Модели потоков данных

Три главных модели:
- Данные, передаваемые через базы данных;
- Данные, передаваемые через сервисы, как запросы API
- Данные, передаваемые через потоки в реальном времени(Apache Kafka)

#### Данные, передаваемые через базы данных

Самый легкий путь передавать данные между двумя процессами. Например, процесс А пишет и читает БД, а процесс В лишь читает. Но такой режим может быть не эффективным, так как он требует, чтобы у обоих процессов был доступ к БД, а также запись/чтение в БД может быть медленным, что не подойдет к системам, где нужна низкая задержка.

#### Данные, передаваемые с помощью сервисов

Другой способ передавать данные между двумя процессами - это отправлять данные напрямую через сеть, которая соединяет эти два процесса. Чтобы передать данные от процесса В в процесс А, процесс А первый отправляет запрос процессу Б, который указывает данные, которые нужны процессу А, и процесс Б возвращает запрашиваемые данные.

Этот режим связан с передачей данных через сервисы. Поэтому эта архитектура является *request-driven*. И является синхронным, т.е. пока будет ожидаться один запрос, система не будет работать дальше. Такая архитектура лучше, если система больше завязана на логике.

#### Данные, передаваемые с помощью брокеров

Вместо запросов данных между сервисами напрямую, каждый сервис может коммуницировать с брокером

![[Pasted image 20260304194306.png]]

Технически БД может быть брокером, но для приложений с низкой задержкой она не подходит. Кусок трансляции данных называется *событием*. А сама архитектура *event-driven*. Она лучше подходит для систем с огромным количеством данных.
Два самых известных типа: pubsub и очередь сообщений.

### Поточная обработка или обработка кусками

*Потоковая обработка* происходит сразу по мере поступления данных, объединяя новые вычисления со старыми. *Обработка наборами* наоборот обрабатывает данные уже с источников данных(БД и тд.). 

Потоковая более эффективна, так как нам не нужно сначала записывать данные в БД. В ML чаще используется именно она, обработка наборами используется для вычисления признаков, которые меняются не так часто. Признаки наборов - признаки, извлеченные через обработку наборами также называются *статическими*.

Потоковая обработка используется для признаков, который меняются быстро. Потоковые признаки называются *динамическими*.
